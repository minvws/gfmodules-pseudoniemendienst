# Quality Assurance release checklist

This checklist should be copied into an issue when a release candidate tag is created. For consistency, please copy the
title and fill in the correct version number.

---

## QA Release Checklist (v4.2.4-rc)

- [ ] Check: code coverage meets specifications
- [ ] Test new logging items and inform SOC SIEM/CSIRT
  - [ ] Notify relevant SOC and/or functional administrators
- [ ] Discuss changes/issues with QA to ensure accurate/rapid testing
- [ ] Test report created ([See template](https://github.com/minvws/nl-rdo-quality-coordination/blob/main/Kwaliteitsmanagement/Kwaliteitsaanpak/Testrapport/Testrapport.md))
- [ ] Arrange any required post-release tests and/or monitoring.
